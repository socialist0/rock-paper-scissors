# 미니게임 프로젝트 개발 로그 (Project.md)

## 📌 프로젝트 개요
Supabase 백엔드와 GitHub Pages 프론트엔드를 연동한 실시간 명예의 전당 기능 탑재 미니게임 플랫폼입니다. 보안 위변조 방지 및 악의적인 트래픽 도배를 차단하는 상용 서비스 수준의 보안 레이어가 적용되어 있습니다.

---

## 🎮 포함된 게임 및 특징

### 1. 👊 가위바위보 (RPS Game)
- **게임 방식:** 컴퓨터와 실시간 가위바위보 대결을 펼쳐 연속 승리(연승) 기록을 측정합니다.
- **밸런스 패치:** 게임 진행 속도가 빠른 특성을 고려하여 **1초의 연속 제출 제한(쿨타임)**이 적용되어 있으며, 사용자 경험을 위해 경고창(alert) 없이 부드럽게 연타가 차단됩니다.

### 2. 🎯 원 그리기 (Circle Drawing Game)
- **게임 방식:** HTML5 Canvas 위에 마우스나 터치로 정밀하게 원을 그려 완벽한 원에 가까운지 정확도(%)를 판정합니다.
- **판정 알고리즘:**
  1. 원의 중심점(X, Y 평균) 및 평균 반지름 도출
  2. 모양의 일관성 계산 (표준편차 감점)
  3. 시작점과 끝점의 벌어진 틈새(Gap 페널티) 계산
  4. 가장 많이 튀어나오거나 들어간 곳(최대 오차 비율 페널티) 계산
- **밸런스 패치:** 정성껏 그리는 게임 특성 및 스팸 방지를 위해 **사용자 지정 쿨타임(기본 20초에서 필요에 따라 5초 등으로 단축 가능)**이 적용되며, 쿨타임 중 재제출 시 남은 시간이 경고창으로 안내됩니다.

---

## 🛡️ 적용된 보안 시스템 (Security Architecture)

현재 프로젝트는 `index.html`, `script.js` 외에 별도의 보안 전용 레이어인 **`security.js`**를 도입하여 이중 잠금 체계를 갖추고 있습니다.

### 1. 데이터 위변조 방지 (Crypto Hash Verification)
- **원리:** 점수를 Supabase 서버로 전송하기 직전, 클라이언트 사이드에서 유저의 닉네임과 점수, 그리고 외부인은 알 수 없는 나만의 비밀 키(`SECRET_SALT`)를 조합하여 `SHA-256` 암호화 해시 토큰을 생성합니다.
- **데이터베이스 연동:** Supabase 테이블에 `verification_token` 컬럼을 신설하여, 점수 등록 시 이 암호화 토큰을 함께 탑재해 전송합니다. 이를 통해 추후 서버나 관리자 단에서 비정상적인 경로로 찔러 넣은 가짜 점수를 원천 필터링할 수 있습니다.

### 2. 로컬 스토리지 기반 트래픽 도배 방지 (Time-Lock)
- **원리:** 브라우저의 `localStorage` 시스템을 이용해 성공적으로 점수가 저장된 시점의 타임스탬프를 기억합니다.
- **게임별 독립 제한 (독립 쿨타임 시스템):**
  - 가위바위보 제출 타임락 키: `last_submit_time_rps` (1초 제한)
  - 원 그리기 제출 타임락 키: `last_submit_time_circle` (사용자 설정 초 제한)
- 유저가 매크로 프로그램을 이용해 비정상적인 연타를 가하더라도 `script.js` 진입 입구에서 차단되어 Supabase API 호출(트래픽 비용)을 원천 봉쇄합니다.

## 📂 프로젝트 파일 구조
```text
내-프로젝트-폴더/
├── index.html          # 웹 게임 UI 레이아웃 및 스크립트 로드 (security.js -> script.js 순서)
├── script.js           # 게임 비동기 로직, Canvas 판정 알고리즘, Supabase 데이터 전송 담당
├── security.js         # 암호화 해시 생성(SHA-256) 및 게임별 로컬스토리지 쿨타임 통제 담당
├── project.md          # [현재 파일] 프로젝트 개발 명세 및 히스토리 관리 문서
└── ment.json           # 원 그리기 결과에 따라 출력되는 20단계 세부 평가 멘트 데이터셋

 Project: 미니게임 플랫폼 고도화 개발 명세서본 프로젝트는 Supabase 백엔드 연동 및 실시간 랭킹 시스템을 갖춘 반응형 웹 미니게임 플랫폼입니다. 보안 무결성 확보와 소스코드의 유지보수성 향상을 위해 확장형 모듈 구조(Modular Architecture)로 전면 개편되었습니다.📂 
 
 1. 최신 프로젝트 파일 구조새로운 미니게임이 추가되더라도 기존 코드를 수정하지 않고 플러그인처럼 쉽게 결합할 수 있도록 기능을 파일별로 완전 격리했습니다.index.html역할: 전체 플랫폼의 UI 레이아웃, Canvas 구성, 탭 전환 메뉴 및 외부 라이브러리(Supabase, CryptoJS) 로드 관리.특이사항: 스크립트 간 의존성을 고려하여 security.js -> main.js -> 각 게임 모듈 순으로 로드하도록 정렬..gitignore역할: 로컬 개발 환경의 백업 파일(scriptold.js, 날짜별 백업본 등)이 GitHub 원격 저장소에 업로드되지 않도록 차단하여 소스 관리 효율화.security.js역할: 클라이언트 사이드 보안 컨트롤러.주요 기능: 점수 위변조 방지를 위한 SHA-256 기반 암호화 해시(verification_token) 생성 및 게임별 독립형 디바이스 타임락(가위바위보 1초 / 원 그리기 20초 쿨타임) 가동.main.js역할: 플랫폼 공통 관리자 모듈.주요 기능: Supabase 클라이언트 초기화 및 글로벌 공유(window._supabase), 유저 세션 관리(닉네임 로그인), 게임 전환 탭 바 통제, 외부 ment.json 비동기 프리로딩 캐시 구축.game_rps.js역할: '가위바위보' 게임 전용 독립 모듈.주요 기능: 난수 기반 컴퓨터 패 생성, 실시간 연승(Streak) 연산, rankings 테이블 점수 업로드 및 글로벌 실시간 TOP 10 랭킹 렌더링.game_circle.js역할: '원 그리기' 게임 전용 독립 판정 모듈.주요 기능: HTML5 Canvas 마우스/터치 드로잉 이벤트 트래킹, 초엄격 원 그리기 판정 엔진 구동, circle_rankings 테이블 업로드 및 명예의 전당 랭킹 출력.ment.json역할: 원 그리기 최종 점수대별(0% ~ 100%) 맞춤형 평가 피드백 문구가 담긴 정적 데이터 파일.🎯 2. 고도화된 '원 그리기' 판정 알고리즘 명세단순히 전체 점들의 평균 반지름과 표준편차만 비교하던 기존 방식에서 탈피하여, 지글거림과 미시적 일탈을 잡아내는 3중 감점 시스템을 적용하여 판정의 엄격함을 극대화했습니다.1) 기본 도형 점수 (shapeScore)원리: 사용자가 그린 모든 좌표의 중심점(X, Y 평균)을 계산한 뒤, 각 좌표와 중심점 사이의 거리(반지름)들의 표준편차를 구합니다.수식: shapeScore = (1 - (반지름 표준편차 / 평균 반지름) * 2) * 1002) 미시적 요동 및 지글거림 감점 (smoothnessPenalty)목적: 매크로 프로그램 사용이나 손떨림으로 인해 선이 부르르 떨리며 지그재그로 그려지는 행위를 완벽하게 차단합니다.원리: 선이 그려진 경로를 따라가며 바로 인접한 두 점 사이의 반지름 격차(Math.abs(distances[i] - distances[i - 1]))를 모두 누적 계산합니다. 선이 매끄럽지 않고 찌글찌글할수록 누적 오차가 기하급수적으로 늘어납니다.수식 가중치: 계산된 요동 비율에 250의 높은 가중치를 곱해 최종 점수에서 다이렉트 감점합니다.3) 돌출 및 함몰 최대 오차 감점 (maxErrorPenalty)목적: 전체적으로 원 모양을 잘 유지했더라도, 특정 한 구역이 툭 튀어나오거나 찌그러진 형태(감자 모양)를 감지해 냅니다.원리: 전체 반지름 리스트 중 평균 반지름과 비교했을 때 가장 격차가 큰 단 하나의 '최대 오차값(Max Error)'을 찾아내어 저격 감점을 진행합니다.수식 가중치: (최대 오차 / 평균 반지름) * 40 만큼 추가 페널티 부여.4) 시작점-끝점 틈새 감점 (gapPenalty)목적: 원을 그리다 말거나 완전히 맞물려 닫히지 않은 미완성 도형을 감점합니다.원리: 첫 번째 좌표와 마지막 좌표 사이의 직선거리(Gap)를 측정합니다.수식 가중치: (벌어진 거리 / 평균 반지름) * 25 만큼 감점.📊 최종 점수 산출 공식$$\text{Final Score} = \text{shapeScore} - \text{gapPenalty} - \text{maxErrorPenalty} - \text{smoothnessPenalty}$$(최종 결과는 $0$점 ~ $100$점 사이로 보정되며, 소수점 첫째 자리까지 반올림하여 표기합니다.)🚀 3. 향후 확장 가이드 (New Game 플러그인 추가 방법)아키텍처가 완전히 분리되었으므로, 나중에 새로운 게임(예: 세 번째 미니게임 game_dino.js)을 추가할 때는 기존 핵심 코드를 한 줄도 건드릴 필요가 없습니다.화면 설계: index.html에 새로운 게임이 들어갈 컨텐츠 구역(<div id="content-dino">)과 상단 탭 버튼을 하나 추가합니다.모듈 생성: game_dino.js 파일을 새로 만들고 그 안에 게임 루프 로직과 점수 업로드 함수(uploadDinoScore)를 작성합니다.안전 연결: index.html 최하단에 <script src="game_dino.js"></script>를 새롭게 한 줄 링크해 주면, main.js와 security.js에 내장된 Supabase 클라이언트 및 보안 해시 엔진을 그대로 가져다 쓸 수 있습니다.최종 업데이트 완료일: 2026년 5월 21일상태: 안정성 검증 완료 및 파일 분리 성공 (정상 작동 중)
