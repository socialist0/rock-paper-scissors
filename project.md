미니게임 프로젝트 개발 로그 (Project.md)
📌 프로젝트 개요
Supabase 백엔드와 GitHub Pages 프론트엔드를 연동한 실시간 명예의 전당 기능 탑재 미니게임 플랫폼입니다. 보안 위변조 방지 및 악의적인 트래픽 도배를 차단하는 상용 서비스 수준의 보안 레이어가 적용되어 있으며, 확장형 모듈 구조(Modular Architecture)로 설계되어 있습니다.

🎮 포함된 게임 및 플랫폼 주요 특징
1. 👊 가위바위보 (RPS Game)
게임 방식: 컴퓨터와 실시간 가위바위보 대결을 펼쳐 연속 승리(연승) 기록을 측정합니다.

밸런스 패치: 게임 진행 속도가 빠른 특성을 고려하여 1초의 연속 제출 제한(쿨타임)이 적용되어 있으며, 사용자 경험을 위해 경고창(alert) 없이 부드럽게 연타가 차단됩니다.

2. 🎯 원 그리기 (Circle Drawing Game)
게임 방식: HTML5 Canvas 위에 마우스나 터치로 정밀하게 원을 그려 완벽한 원에 가까운지 정확도(%)를 판정합니다.

드로잉 연출: 게이머가 원을 그릴 때는 직관적이고 몰입감 높은 파란색 실선으로 캔버스에 렌더링됩니다.

컴퓨터 매칭 시각화 엔진: 유저가 드로잉을 끝내고 마우스/터치를 떼는 순간, 판정 엔진이 계산한 실시간 수학적 중심점(무게중심)에 빨간색 점이 찍히고, 그 중심점과 평균 반지름을 기준으로 삼는 컴퓨터의 완벽한 정원이 '빨간색 점선 가이드라인'으로 위에 부드럽게 겹쳐서 나타납니다. 이를 통해 게이머는 자신이 어느 구간을 찌그러트렸는지 시각적으로 처절하게(?) 체감할 수 있어 극대화된 중독성과 승부욕을 제공합니다.

중심점 산출 원리: 게이머가 붓질을 하며 화면에 남긴 모든 좌표(X, Y)들의 수학적 평균 위치(Centroid, 무게중심)를 추적하여 중심을 잡습니다. 따라서 한쪽 구역을 찌그러트리거나 꼬리를 길게 늘어뜨리면 중심점 자체가 그쪽으로 이동하여 전체적인 가이드라인 불일치가 일어납니다.

밸런스 패치: 95점 이상 획득 시 깜짝 축하 페이지(suddenwinner.html)로 이동하는 히든 보상이 작동합니다.

🛡️ 적용된 보안 및 플랫폼 백엔드 아키텍처
1. 데이터 위변조 방지 (Crypto Hash Verification)
원리: 점수를 Supabase 서버로 전송하기 직전, 클라이언트 사이드에서 유저의 닉네임과 점수, 그리고 외부인은 알 수 없는 나만의 비밀 키(SECRET_SALT)를 조합하여 SHA-256 암호화 해시 토큰을 생성합니다.

데이터베이스 연동: Supabase 테이블에 verification_token 컬럼을 활용하여 점수 등록 시 이 암호화 토큰을 함께 탑재해 전송합니다. 이를 통해 추후 서버나 관리자 단에서 비정상적인 경로로 찔러 넣은 가짜 점수를 원천 필터링할 수 있습니다.

2. 로컬 스토리지 기반 트래픽 도배 방지 (Time-Lock)
원리: 브라우저의 localStorage 시스템을 이용해 성공적으로 점수가 저장된 시점의 타임스탬프를 기억하여 Supabase API 트래픽 비용을 보호합니다.

3. 초효율적 데이터 청소 시스템 (Daily Cron Job & Top 10 보존)
원리: 유저가 하루에 수천, 수만 번을 플레이하더라도 Supabase의 저장 용량 한계(Free 플랜 500MB 등)나 성능 저하 문제를 원천 차단하기 위해 매일 하루에 한 번, 상위 10개(Top 10) 기록만 남기고 나머지는 자동으로 삭제하는 SQL 크론탭(pg_cron) 시스템이 백엔드 내부에서 가동 중입니다.

보안 무결성 보완: GitHub Pages 호스팅 구조상 Supabase 공개용 익명 키(anon_key)가 프론트엔드 코드(config.js)에 무조건 포함되어 외부에 노출되지만, Supabase 백엔드 내부의 RLS(Row Level Security, 행 레벨 보안) 시스템을 철저히 활성화해 두었기 때문에 제3자가 키를 탈취하더라도 데이터 임의 삭제나 변조가 원천 차단되므로 구조적으로 매우 안전합니다.

📂 프로젝트 파일 구조 및 연동 명세
index.html : 웹 게임 UI 레이아웃, Canvas 구성, 헤더 통합 및 스크립트 로드 총괄

style.css : 플랫폼 공통 레이아웃 및 탭 메뉴, 버튼, 랭킹 리스트 디자인 스타일시트

security.js : 암호화 해시 생성(SHA-256) 및 게임별 쿨타임 타임락 통제 담당

main.js : 플랫폼 공통 관리자 모듈. Supabase 초기화, 닉네임 세션 및 타이틀 네비게이션 통제

game_rps.js : 가위바위보 독립 모듈 (컴퓨터 패 산출, 연승 연산, 랭킹 데이터 입출력)

game_circle.js : 원 그리기 독립 모듈 (파란 실선 드로잉, 빨간 점선 가이드 매칭 시각화, 판정 엔진)

ment.json : 원 그리기 최종 점수대별(0% ~ 100%) 맞춤형 평가 피드백 멘트 데이터셋 (20단계)

project.md : [현재 파일] 프로젝트 개발 명세 및 전체 히스토리 관리 가이드 문서

🚨 파일 간 핵심 연동 및 트러블슈팅 히스토리
스크립트 로드 순서 의존성: 반드시 security.js -> main.js -> 각 게임 모듈 순서로 로드되어야 Supabase 클라이언트 및 암호화 해시 엔진을 에러 없이 상속받을 수 있습니다.

글로벌 헤더 링크 연동 ("잠시만 혹시...?"):

메인 타이틀을 "잠시만 혹시...?"라는 텍스트로 변경하고, 게임 진행 중 타이틀을 클릭하면 언제든지 최초 닉네임 설정 화면으로 복귀하는 내비게이션 링크를 구현했습니다.

자바스크립트(main.js)의 비동기 로드 타이밍 및 요소 탐색 오류를 차단하기 위해 window.addEventListener('load', ...) 기반의 초강력 방어형 이벤트를 바인딩했으며, index.html 내의 실제 레이아웃 ID 명찰 구조인 #user-setup 및 #game-area와 데이터 싱크를 100% 매치 완료하여 콘솔 에러를 완전 차단했습니다.

📊 고도화된 '원 그리기' 판정 알고리즘 상세 공식
단순히 전체 점들의 평균 반지름과 표준편차만 비교하던 방식에서 탈피하여, 지글거림과 미시적 일탈을 잡아내는 3중 감점 시스템을 적용하여 판정의 엄격함을 극대화했습니다.

기본 도형 점수 (shapeScore)

사용자가 그린 모든 좌표의 중심점(X, Y 평균 위치 = 무게중심)을 계산한 뒤, 각 좌표와 중심점 사이의 거리(반지름)들의 표준편차를 구합니다.

공식: shapeScore = (1 - (반지름 표준편차 / 평균 반지름) * 2) * 100

미시적 요동 및 지글거림 감점 (smoothnessPenalty)

매크로 프로그램 사용이나 손떨림으로 인해 선이 부르르 떨리며 지그재그로 그려지는 행위를 차단합니다.

선이 그려진 경로를 따라가며 바로 인접한 두 점 사이의 반지름 격차(Math.abs(distances[i] - distances[i - 1]))를 모두 누적 계산합니다. 선이 매끄럽지 않고 찌글찌글할수록 누적 오차가 기하급수적으로 늘어납니다. (수식 가중치: 계산된 요동 비율에 250을 곱해 다이렉트 감점)

돌출 및 함몰 최대 오차 감점 (maxErrorPenalty)

전체적으로 원 모양을 잘 유지했더라도, 특정 한 구역이 툭 튀어나오거나 찌그러진 형태(감자 모양)를 정밀 타격합니다.

전체 반지름 리스트 중 평균 반지름과 비교했을 때 가장 격차가 큰 단 하나의 '최대 오차값(Max Error)'을 찾아내어 저격 감점을 진행합니다. (수식 가중치: (최대 오차 / 평균 반지름) * 40 만큼 페널티 부여)

시작점-끝점 틈새 감점 (gapPenalty)

원을 그리다 말거나 완전히 맞물려 닫히지 않은 미완성 도형을 감점합니다. 첫 번째 좌표와 마지막 좌표 사이의 유클리드 직선거리(Gap)를 측정합니다. (수식 가중치: (벌어진 거리 / 평균 반지름) * 25 만큼 감점)

📈 최종 점수 산출 공식
최종 스코어 = shapeScore - gapPenalty - maxErrorPenalty - smoothnessPenalty

(최종 결과는 0점 ~ 100점 사이로 보정되며, 소수점 첫째 자리까지 반올림하여 표기하고 랭킹 데이터베이스에 업로드됩니다.)
